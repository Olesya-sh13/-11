Как работает шифрование логов?

Критически важные события безопасности (попытки злоумышленника, нарушения политики, аномалии частоты) сохраняются не в открытом виде, а в зашифрованном формате. Это означает, что даже если кто-то получит доступ к файловой системе сервера, он увидит только нечитаемый набор символов — без ключа расшифровки данные бесполезны.

Используется алгоритм Fernet (AES-128-CBC + HMAC-SHA256) — симметричное шифрование из библиотеки cryptography. Ключ хранится отдельно от данных и никогда не передаётся по сети. Расшифровка возможна только из этого защищённого административного интерфейса.

1
Событие зафиксировано
Система обнаруживает инцидент: попытку взлома, нарушение политики или аномальную активность
2
Данные шифруются
JSON с деталями события кодируется ключом Fernet и сохраняется в файл .enc
3
Файл нечитаем
Без ключа файл выглядит как случайный набор символов — Base64 от зашифрованных байт
4
Только admin видит содержимое
Нажав «Расшифровать», администратор получает читаемую таблицу событий
